# CodePulse 故障排查

## 额度暂不可用

当数据源无法提供可靠额度时，界面显示“额度暂不可用”。此状态不能显示为 0%，也不能触发额度耗尽提示。

## 数据过期

运行中任务超过过期时间后会转为“数据过期”，不再继续显示为“运行中”。需要刷新数据源或检查适配器状态。

## 托盘不可见

1. 确认应用仍在运行。
2. 检查 Windows 托盘隐藏图标区域。
3. 如果刚重启过 Explorer，应用会在检测到 explorer.exe 进程 ID 变化后尝试重建托盘图标。
4. 如果重建失败，主进程会记录“托盘图标重建失败：具体原因”，后续诊断导出可继续补充该状态。

## 弹窗位置异常

贴边弹窗会根据托盘图标、鼠标点和显示器工作区计算位置。多显示器或自动隐藏任务栏场景下，如果无法判断方向，会按底部任务栏处理并限制在工作区内。

显示器热插拔或 DPI 指标变化时，主进程会重新应用动态岛位置并关闭贴边弹窗。再次点击托盘图标会按当前显示器重新计算弹窗位置。

## 系统休眠后状态异常

应用监听系统休眠和恢复事件。休眠前会关闭贴边弹窗，恢复后会重新应用动态岛位置并刷新 AgentStateHub 状态。如果恢复后仍显示过期数据，可手动刷新状态或检查数据源连接状态。

## 历史数据库损坏

历史库加载时会执行 SQLite 完整性检查。检测到损坏后，应用会将原 `history.sqlite` 重命名为 `.corrupt-时间戳` 备份文件，并重建空历史库以避免启动失败。诊断导出会包含历史库是否发生过损坏恢复、备份路径和最近清理记录，路径会经过脱敏处理。

## IPC 调用失败

渲染进程只能通过 `window.codePulse` 调用白名单 API。若页面提示 IPC 调用失败，需要检查主进程是否完成 IPC 注册，以及 Preload 是否正确加载。

## 打开 Agent 失败

Codex 任务会在任务项目目录中通过受控 PowerShell 命令启动 Codex。Process、Log、CustomCommand 和 Mock 等无法恢复具体会话的数据源会回退打开任务项目目录。如果提示“Agent 启动目录暂不可用”或“Agent 项目目录暂不可用”，说明该任务没有可用项目路径；如果提示“Agent 项目目录打开失败”，需要检查目录是否存在以及当前用户是否有访问权限。

## 诊断导出包含敏感信息

诊断导出会在主进程侧脱敏 Windows 用户目录、完整路径、API Key、Token、Authorization 和敏感命令参数。若仍发现敏感内容，应先停止分享该诊断文件，再补充对应脱敏规则。
